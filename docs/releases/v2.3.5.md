### System Health Debugging

**Problem:** System health metrics showing 0.0% or empty values with no way to diagnose why.

**Solution:** 
- Added comprehensive logging for system health updates
- Logs warnings when paths don't exist
- Logs errors when disk usage calculation fails
- Logs debug info for successful updates (visible with `--log-level DEBUG`)

**Impact:** Easier to diagnose system health issues. Logs will show exactly why disk space isn't being calculated.

### Countdown Visible from Start

**Problem:** Countdown for next run was only visible after the first run completed, leaving users wondering when the next run would start.

**Solution:** 
- Set initial `next_run_time` when application starts
- Countdown now shows immediately when repeat mode is enabled
- No need to wait for first run to complete

**Impact:** Better user experience - countdown visible from application start.

### Next Run Time Set After Processing

**Problem:** After processing completed, the next run countdown wasn't always visible.

**Solution:** 
- Re-set `next_run_time` after each processing run completes
- Ensures countdown is always visible between runs when repeat mode is enabled

**Impact:** Countdown updates correctly and is always visible when repeat mode is active.

```python
    tracker.set_next_run(settings.repeat_after_minutes)
```

### Next Run Time Reset

```python
# Set next run time after processing completes (if repeat mode is enabled)
if args.webgui and settings.repeat_forever and settings.repeat_after_minutes > 0:
    tracker.set_next_run(settings.repeat_after_minutes)
```

## üìã Migration Guide

### No Migration Required

This release is fully backward compatible. No migration steps are required.

### Debugging System Health Issues

If system health still shows 0.0%, check logs:

1. Enable debug logging: `--log-level DEBUG`
2. Look for log messages containing "System Health:"
3. Check if paths exist and are accessible
4. Verify Docker volume mounts are correct
