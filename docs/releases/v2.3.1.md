### System Health Updates Fixed

**Problem:** System information (disk space, CPU usage, memory usage) was not displayed in the WebGUI, showing empty values or "0%".

**Root Cause:** System health metrics were only updated once at application startup, not during runtime.

**Solution:** 
- System health metrics are now updated every 30 seconds during idle periods
- Updates occur continuously in the sleep loop between runs
- All metrics (disk space for Downloads/Extracted/Finished, CPU, memory, 7-Zip version) are refreshed regularly

**Impact:** Users can now see real-time system information in the WebGUI, including:
- Disk space usage for all three directories (Downloads, Extracted, Finished)
- CPU and memory usage percentages
- 7-Zip version information

## üîß Technical Details

- Added periodic `update_system_health()` calls in the main sleep loop
- Updates occur every 30 seconds when the application is idle
- No performance impact as updates are lightweight and non-blocking
