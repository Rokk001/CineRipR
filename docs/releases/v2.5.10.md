### TemplateNotFound: index.html - Root Cause Fixed

**Problem:**
- `jinja2.exceptions.TemplateNotFound: index.html` persistierte trotz absolutem Pfad
- Flask konnte das Template `index.html` nicht finden
- 500 Internal Server Error beim Zugriff auf `/`

**Root Cause:**
- Templates und Static Files wurden **nicht im installierten Paket mitgeliefert**
- `pyproject.toml` hatte keine `package-data` Konfiguration
- Beim Build wurden nur Python-Dateien (.py) ins Wheel-Paket aufgenommen
- Templates (`templates/`) und Static Files (`static/`) fehlten im installierten Paket

**Solution:**
- `[tool.setuptools.package-data]` Konfiguration in `pyproject.toml` hinzugefÃ¼gt
- Templates und Static Files werden jetzt explizit im Paket enthalten:
  ```toml
  [tool.setuptools.package-data]
  "cineripr.web" = ["templates/**/*", "static/**/*"]
  ```

**Impact:**
- Templates und Static Files werden jetzt korrekt im installierten Paket enthalten
- WebGUI lÃ¤dt jetzt korrekt in Docker-Umgebung
- Keine 500-Fehler mehr beim Zugriff auf `/`
