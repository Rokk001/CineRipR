### AttributeError in StatusTracker Fixed

**Problem:**
- `AttributeError: 'StatusTracker' object has no attribute 'get_seconds_until_next_run'`
- In `status.py` Zeile 837 wurde `self.get_seconds_until_next_run()` aufgerufen
- Die Methode existiert aber in `GlobalStatus` (Zeile 118), nicht in `StatusTracker`

**Root Cause:**
- Falscher Methodenaufruf in `set_next_run()`
- `get_seconds_until_next_run()` ist eine Methode von `GlobalStatus`, nicht von `StatusTracker`

**Solution:**
- `self.get_seconds_until_next_run()` ‚Üí `self._status.get_seconds_until_next_run()`
- Debug-Logging entfernt

**Impact:**
- App startet jetzt ohne AttributeError
- `set_next_run()` funktioniert korrekt

### TemplateNotFound: index.html Fixed

**Problem:**
- `jinja2.exceptions.TemplateNotFound: index.html`
- Flask konnte das Template `index.html` nicht finden
- 500 Internal Server Error beim Zugriff auf `/`

**Root Cause:**
- Relativer Template-Pfad `'templates'` funktioniert nicht in Docker-Umgebung
- Flask sucht relativ zum `__name__` (`cineripr.web`), aber der Pfad ist nicht korrekt

**Solution:**
- Template-Pfad jetzt absolut mit `Path(__file__).parent / 'templates'`
- Static-Folder-Pfad jetzt absolut mit `Path(__file__).parent / 'static'`
- Flask findet jetzt Templates und Static Files korrekt

**Impact:**
- WebGUI l√§dt jetzt korrekt
- Keine 500-Fehler mehr beim Zugriff auf `/`
