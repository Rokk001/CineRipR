### Reduced Idle CPU Usage

**Problem:**
- Even after removing the system health background thread, idle CPU usage remained at ~8%
- Settings check was running every 5 seconds during sleep phase
- This caused ~12 database queries per minute even when idle
- Minute-based logging checked the condition every second

**Solution:**
- Increased settings check interval from 5 to 30 seconds
- Reduces database queries from ~12/min to ~2/min
- Optimized minute-based logging to only check once per minute instead of every second
- Idle CPU usage should now drop below 1% when waiting for next run
